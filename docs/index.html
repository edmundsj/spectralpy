Common mistakes / Issues¶

Note: When using ‘hann’ windowing, the DC component of the spectrum will not be the same as the mean, due to spectral leakage. Spectral leakage of the DC component does not occur when using boxcar filtering, but does occur when using hann windowing. The total DC power is still the same, but it is spread over multiple bins near zero (~approx 3).

spectral_processing.power_spectrum(data, window='hann', siding='single', amplitude=False)¶

Computes the single-or double-sided power spectrum with different types of filetring (hann, boxcar)

Parameters

  • data – data to be transformed

  • window – window type - hann or box

  • amplitude – Whether to return the power spectrum or the amplitude spectrum

  • siding – ‘single’ or ‘double’ sided spectrum.

spectral_processing.psd_weights(data, frequency, window='hann', siding='single', amplitude=False)¶

Computes the PSD weights for an ideal sinewave of a given frequency

Parameters

  • frequency – Desired frequency to extract

  • data – Time-domain data we want the frequency-domain weights of

  • window – Window type. Options are “hann” or “boxcar”

  • power – Whether to give magnitude squared (if true) or amplitude (if false)
